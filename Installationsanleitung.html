Schnellstart (GitHub)

Linux / macOS

# Auf frischen Systemen zuerst:
apt update && apt install -y curl dos2unix

# Installation starten:
curl -fsSL https://raw.githubusercontent.com/florianschroller-boop/it-helpdesk/main/github-install.sh -o /tmp/install.sh
dos2unix /tmp/install.sh
bash /tmp/install.sh

Der Installer installiert automatisch alle Abhängigkeiten (git, Node.js 20, MySQL) und führt das Setup durch.

Mit Demo-Daten:

curl -fsSL https://raw.githubusercontent.com/florianschroller-boop/it-helpdesk/main/github-install.sh | bash -s -- --demo

In ein bestimmtes Verzeichnis:

curl -fsSL https://raw.githubusercontent.com/florianschroller-boop/it-helpdesk/main/github-install.sh | bash -s -- --dir /opt/helpdesk --demo

Windows

git clone https://github.com/florianschroller-boop/it-helpdesk.git
cd it-helpdesk
node install.js
START.bat
Voraussetzung: Node.js (≥ 18), MySQL/MariaDB und Git müssen installiert sein. Details siehe Abschnitt 2.

3. Installation via GitHub

3.1 Automatisch (empfohlen)

Der GitHub-Installer prüft Voraussetzungen, klont das Repository, installiert Pakete und startet den Setup-Assistenten:

# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/florianschroller-boop/it-helpdesk/main/github-install.sh | bash

# Windows (Git erforderlich)
git clone https://github.com/florianschroller-boop/it-helpdesk.git
cd it-helpdesk
node install.js

3.2 Manuell

Repository klonen:
git clone https://github.com/florianschroller-boop/it-helpdesk.git

In den Ordner wechseln:
cd it-helpdesk

Pakete installieren:
npm install --production

Setup-Assistent starten:
node install.js

Hinweis: Der Installer fragt Schritt für Schritt nach Datenbank-Zugangsdaten, Admin-Account und optionalen Demo-Daten. Jeder Schritt wartet auf Bestätigung.

5. Installation unter Linux

Node.js + MySQL installieren

Ubuntu/Debian:

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs mysql-server
sudo systemctl start mysql

CentOS/RHEL:

curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs mariadb-server
sudo systemctl start mariadb

Helpdesk installieren

Option A — Automatisch (installiert alle Abhängigkeiten):

# Falls curl fehlt: apt update && apt install -y curl
curl -fsSL https://raw.githubusercontent.com/florianschroller-boop/it-helpdesk/main/github-install.sh | bash

Option B — Manuell:

cd /opt
git clone https://github.com/florianschroller-boop/it-helpdesk.git helpdesk
cd helpdesk
npm install --production
node install.js
./start.sh

Als systemd-Dienst (optional)

sudo tee /etc/systemd/system/helpdesk.service <<EOF
[Unit]
Description=IT-Helpdesk
After=network.target mysql.service
[Service]
Type=simple
WorkingDirectory=/opt/IT-Helpdesk
ExecStart=/usr/bin/node /opt/IT-Helpdesk/api/index.js
Restart=always
Environment=NODE_ENV=production
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl enable --now helpdesk

6. Plugins installieren

Als Admin anmelden

Administration → Plugins

"Plugin installieren" → ZIP-Datei hochladen

Server neustarten (Admin → Server neustarten)

Tipp: Plugins können auch manuell installiert werden: ZIP entpacken nach plugins/<name>/, dann Server neustarten.
Sichtbarkeit: Im Plugin-Manager können Admins festlegen, welche Plugins für Endbenutzer oder Agents sichtbar sind (Sidebar-Einträge). Die Plugin-Funktionen bleiben über direkte URLs erreichbar.

7. Updates einspielen

Via GitHub (empfohlen)

cd /opt/helpdesk    # oder Ihr Installationsverzeichnis
git pull
npm install --production

Dann Server neustarten (im Browser: Administration → Server neustarten, oder im Terminal).

Was passiert beim Update:
  • Neue Dateien werden heruntergeladen, bestehende aktualisiert
  • Neue npm-Pakete werden installiert
  • Neue Datenbank-Migrationen werden beim nächsten Server-Start automatisch erkannt
  • Plugin-Daten und .env bleiben unberührt

Via Systemwartungs-Plugin

Falls das system-maintenance Plugin installiert ist:

Admin → Systemwartung → "Update einspielen"

Update-ZIP hochladen (erstellt automatisch ein Backup)

Server neustarten

9. Problembehandlung

"Cannot connect to database"
MySQL läuft nicht oder Zugangsdaten in .env falsch. Bei XAMPP: MySQL-Modul starten.
"EADDRINUSE"
Port belegt. In .env den APP_PORT ändern, oder den anderen Prozess beenden.
Plugin erscheint nicht
Server neustarten. Plugins werden beim Start geladen und Frontend-Assets synchronisiert.
Server neustarten
Im Browser: Administration → Server neustarten. Oder: Terminal schließen, START.bat erneut starten.
